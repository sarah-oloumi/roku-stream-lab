# Architecture Decision Records

Use ADRs for decisions that shape the emulator architecture or create long-lived constraints.

Create a new ADR from `template.md` when a story makes a decision about:

- BRS integration strategy.
- SceneGraph parser or renderer architecture.
- Runtime scheduling, fake clock, storage, or network model.
- Apple container workflow.
- Compatibility boundaries where exact Roku behavior is unknown or intentionally unsupported.

ADRs should be short, dated, and linked from the relevant story or compatibility matrix row.
